# 2026-05-21 侧边栏壳层 UI 改造说明

## 背景

当前 popup 采用顶部 tabs + 窄面板布局，在更宽的浏览器侧边栏环境里容易出现内容裁切、信息压缩和“工具壳”感不足的问题。用户希望参考页面代理类扩展的侧边栏样式，形成更稳定的左侧导航 + 右侧内容结构。

## 目标

- 将 popup 重构为桌面侧边栏壳层。
- 让导航、状态和设置入口固定在左侧，右侧承载功能内容。
- 提升页面层级、空状态和配置页的产品感，避免卡片堆叠造成的模板感。
- 保持现有业务逻辑不变，仅调整壳层和展示结构。

## 非目标

- 不修改书签分类、清理、AI 连接等业务逻辑。
- 不引入新依赖。
- 不做移动端 App 化重构。

## 设计

### 布局

- 整体采用双栏布局：左侧导航栏宽约 220px，右侧内容区自适应。
- 左侧栏固定品牌、状态和功能入口，右侧仅展示当前功能内容。
- 容器采用白色 surface + 细边框 + 轻阴影，弱化大面积渐变。

### 导航

- 左侧纵向导航项：智能分类、清理书签、AI 配置。
- 当前项高亮使用浅色底和左侧强调条，不依赖重渐变。
- 导航同时承载简短状态文案，减少顶部区域占高。

### 内容页

- 分类页：保留现有空状态和分类结果，但去掉顶部 tabs。
- 清理页：保留检测参数和结果列表，头部改为窄条操作区。
- AI 配置页：改为更紧凑的配置表单，降低顶部介绍块高度。

### 响应式

- 宽屏优先双栏。
- 如果窗口宽度不足，侧边栏退化为窄 rail，导航项保留 icon+短文案。

## 验收标准

- popup 不再出现 380px / 480px 宽度冲突导致的裁切。
- 左侧导航清晰稳定，当前功能可一眼识别。
- AI 配置页内容不再因顶部说明块过高而被挤压。
- 分类、清理、配置三个页面的布局风格一致。
- `npm run typecheck`、`npm run build` 通过。
